# Export / Import Project — Design

**Date:** 2026-06-10
**Status:** Approved

## Goal

Export a project (Group) or a single Feature to a portable `.zip` archive containing a
`manifest.json` (structure) plus per-session markdown handoff summaries, generated by the
agents themselves (LLM summarization). On import, the structure is recreated and each agent
terminal continues from its summary via a startup prompt.

## Scope

- Export of a whole Group ("Export project…" in the group context menu).
- Export of a single Feature ("Export feature…" in a new feature context menu).
- Import via an "Import…" button next to "+ New Project" at the bottom of the sidebar.
- Out of scope: exporting raw transcripts, shell history, review artifacts, PTY state,
  auto-cleanup of imported files, any git/worktree handling.

## Archive format

```
<name>-<YYYY-MM-DD>.zip
├── manifest.json
└── sessions/
    └── <feature>-<terminal>-<shortid>.md   (sanitized names; shortid = first 4 hex of terminal id)
```

`manifest.json`:

```json
{
  "format": "brain-export",
  "version": 1,
  "scope": "group",
  "exportedAt": "2026-06-10T14:30:00Z",
  "group": { },
  "feature": { },
  "sessions": {
    "<originalTerminalId>": { "file": "sessions/auth-claude-3f2a.md", "kind": "claude" },
    "<otherTerminalId>": { "kind": "codex", "error": "summarization timed out" }
  }
}
```

- `scope` is `"group"` or `"feature"`.
- For `scope: "group"`, `group` is the full `Group` object (original ids kept; remap happens
  at import). `feature` is absent.
- For `scope: "feature"`, `feature` is the full `Feature` object and `group` is reduced to
  `{ "name": string, "cwd": string }` (used only if import has to create a group).
- `sessions` maps original terminal id → summary file, or an `error` string when
  summarization failed (export still succeeds).

## Export flow

1. Renderer sends a snapshot of the Group (or Group + Feature) over new IPC `export:run`.
2. Main process:
   - Shows the save dialog **first** so the user can cancel before any summarization runs.
     Default filename `<name>-<date>.zip`.
   - Collects agent terminals (`kind` claude/codex) that have a `sessionId`.
   - Summarizes each session headlessly, max 3 in parallel, 3-minute timeout per session:
     - Claude: `claude -p --resume <sessionId> "<prompt>"` with cwd set to the terminal's
       cwd (Claude resolves sessions per project dir). `-p --resume` forks the session;
       the original is untouched.
     - Codex: `codex exec resume <sessionId> --output-last-message <tmpfile> "<prompt>"` —
       clean output without TUI noise.
   - Summary prompt (both agents): *"Write a handoff summary of this session as markdown,
     in the same language as the conversation: goal, current state, key decisions, files
     touched, and concrete next steps. Output only the markdown."*
   - Emits `export:progress` events (`{ done, total, current }`) to the renderer, shown as
     a small overlay/toast.
   - Builds the zip with `adm-zip` (new dependency) and writes it to the chosen path.
   - Returns `{ ok, path, warnings: string[] }` — one warning per failed summary.
3. Failure of any single summarization (missing CLI, deleted session, timeout, non-zero
   exit) records an `error` in the manifest and a warning in the result; it never aborts
   the export.

## Import flow

1. "Import…" button → open dialog (filter `.zip`).
2. Main (`import:run`): extracts to `userData/imports/<new-uuid>/`, validates
   `manifest.format`/`version`, returns `{ manifest, dir, cwdExists }`. Invalid zip or
   manifest → error notification, workspace untouched.
3. Renderer builds the imported structure:
   - **New ids** for group, features, and terminals (`crypto.randomUUID`) — no collisions
     with the existing workspace.
   - **CWD remap:** if the group cwd does not exist, prompt with a folder picker for a new
     root; terminal cwds that had the old root as prefix are prefix-remapped. Regardless of
     the root check, any terminal cwd that still does not exist after remapping falls back
     to `''` (home).
   - **Continue:** each agent terminal with a summary `.md` gets a `startupCommand`:
     - Claude: `claude --session-id <fresh-uuid> "Read <abs-md-path> — it is a handoff
       summary of a previous session. Continue from there."` (pins a new session id,
       mirroring `agentLaunchCommand`).
     - Codex: `codex "Read <abs-md-path> — …"` plus the existing post-launch session
       capture (`agentCaptureSession`).
   - Shell terminals import with their exported `startupCommand`. Agent terminals
     without a summary (summarization failed) get a REBUILT fresh launch command —
     their exported one may embed a dead `--session-id` or a stale review prompt.
   - Nothing spawns at import time; a session starts when the user opens the terminal.
4. A feature-scope zip imports into the active group; if no group exists, one is created
   from the manifest's `group.name`/`group.cwd`.
5. Extracted files stay permanently under `userData/imports/<id>/` (startup prompts
   reference them; they are small, no auto-cleanup).

## Edge cases

- **ReviewLink:** review paths (`reviewDir`, `transcriptPath`, …) are machine-local, so the
  `review` field is stripped on import; reviewer terminals import as plain agent terminals
  (their sessions are summarized like any other at export).
- **Original `sessionId` is not carried over** on import — continuation goes through the
  `.md` summary, not `--resume`.
- Group/feature name collisions with the existing workspace are allowed (no renaming).

## IPC / API surface

Following the existing pattern (`src/shared/ipc.ts` → `src/main/ipc.ts` →
`src/shared/api.ts` → `src/preload/index.ts`):

- `export:run` (`ipcMain.handle`) — input `{ scope, group, feature? }`, output
  `{ ok, path?, warnings, canceled? }`.
- `export:progress` (`webContents.send`) — `{ done, total, current }`.
- `import:run` (`ipcMain.handle`) — opens dialog, extracts, returns
  `{ manifest, dir, cwdExists } | { canceled } | { error }`.
- New main-process module `src/main/exportImport.ts` holds summarization, zip build,
  extract/validate. Pure remap logic lives in the renderer (e.g.
  `src/renderer/src/importRemap.ts`) so it is unit-testable.

## Dependencies

- `adm-zip` (+ `@types/adm-zip` dev) for archive build/extract. No CLI `zip`/`unzip`
  dependency (Windows builds exist).

## Testing

Vitest, following existing patterns (`persistence.test.ts`, `codexSession.test.ts`):

- Manifest build + zip round-trip through `adm-zip` in a temp dir.
- Summarization spawn logic with mocked `child_process`: command lines for claude/codex,
  timeout → warning, non-zero exit → warning.
- Import remap as a pure function: fresh ids, cwd prefix remap, startupCommand builder,
  review-link stripping, fallback to home for dead cwds.
- Sidebar: new context menu items and Import button (testing-library, existing pattern).
